This endpoint creates a new customer in the Aptly system and automatically generates their first API key. This is the **bootstrap endpoint** that allows admins to onboard new customers.

## Authentication

<ParamField header="X-Admin-Secret" type="string" required>
  The admin secret configured in your `APTLY_ADMIN_SECRET` environment variable
</ParamField>

<Warning>
  This endpoint requires admin authentication. Keep your admin secret secure and never expose it to customers.
</Warning>

## Request Body

<ParamField body="email" type="string" required>
  Customer's email address (must be unique)
</ParamField>

<ParamField body="company_name" type="string">
  Company or organization name
</ParamField>

<ParamField body="plan" type="string" required>
  Subscription plan: `free` or `pro`

  - **free**: 100 requests/hour rate limit
  - **pro**: 1000 requests/hour rate limit
</ParamField>

<ParamField body="compliance_frameworks" type="array">
  List of compliance frameworks this customer needs to adhere to

  Supported values: `["HIPAA", "SOC2", "GDPR", "PCI-DSS"]`
</ParamField>

## Response

<ResponseField name="customer" type="object" required>
  The created customer object

  <Expandable title="customer">
    <ResponseField name="id" type="string">
      Unique customer identifier (e.g., `cus_abc123`)
    </ResponseField>

    <ResponseField name="email" type="string">
      Customer's email address
    </ResponseField>

    <ResponseField name="company_name" type="string">
      Company name
    </ResponseField>

    <ResponseField name="plan" type="string">
      Subscription plan (`free` or `pro`)
    </ResponseField>

    <ResponseField name="compliance_frameworks" type="array">
      List of compliance frameworks
    </ResponseField>

    <ResponseField name="retention_days" type="integer">
      Audit log retention period (default: 2555 days / ~7 years)
    </ResponseField>

    <ResponseField name="pii_redaction_mode" type="string">
      Default PII redaction mode (`mask`, `hash`, or `remove`)
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api_key" type="object" required>
  The generated API key (only returned once)

  <Expandable title="api_key">
    <ResponseField name="id" type="string">
      Unique key identifier
    </ResponseField>

    <ResponseField name="key" type="string">
      **The full API key** (e.g., `apt_live_xyz789...`)

      <Warning>
        This is the **only time** the full key is returned. Store it securely and provide it to the customer. It cannot be retrieved again.
      </Warning>
    </ResponseField>

    <ResponseField name="key_prefix" type="string">
      First 12 characters of the key (for identification)
    </ResponseField>

    <ResponseField name="name" type="string">
      Key name (default: "Default API Key")
    </ResponseField>

    <ResponseField name="rate_limit_per_hour" type="integer">
      Requests per hour allowed for this key
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of key creation
    </ResponseField>
  </Expandable>
</ResponseField>

```bash cURL

  -H "X-Admin-Secret: your-admin-secret" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@company.com",
    "company_name": "Acme Corporation",
```

```python Python
import requests

response = requests.post(
    "https://api.aptly.dev/v1/admin/customers",
    headers={
        "X-Admin-Secret": "your-admin-secret",
        "Content-Type": "application/json"
    },
    json={
        "email": "admin@company.com",
        "company_name": "Acme Corporation",
        "plan": "pro",
        "compliance_frameworks": ["HIPAA", "SOC2"]
    }

result = response.json()

# CRITICAL: Save this API key - it won't be shown again!
api_key = result["api_key"]["key"]

# Provide the API key to the customer securely
```

## Best Practices

### Security

1. **Protect the admin secret** - Never commit it to version control
2. **Use environment variables** - Store in Railway/Heroku config, not code
3. **Limit admin access** - Only trusted team members should have the admin secret
4. **Rotate the secret** - Change it periodically and after team member departures

### API Key Handling

<Warning>
  The full API key is only returned **once** during customer creation. You must:

  1. Capture it from the response
  2. Store it securely (e.g., password manager, secrets vault)
  3. Provide it to the customer via secure channel (not email!)
  4. The customer can create additional keys later via `/v1/api-keys`
</Warning>

**Recommended Flow:**

```python
# Create customer
result = create_customer(email="admin@company.com", plan="pro")
api_key = result["api_key"]["key"]

# Store in your secrets management system
secrets_manager.store(f"aptly_key_{result['customer']['id']}", api_key)

# Send to customer via secure channel
send_secure_message(
    to=result["customer"]["email"],
    subject="Your Aptly API Key",
    message=f"Your API key: {api_key}\n\nStore this securely!"
)
```

### Plan Selection

Choose the appropriate plan based on customer needs:

| Plan | Rate Limit | Use Case |
|------|------------|----------|
| **free** | 100 req/hour | Development, testing, small projects |
| **pro** | 1000 req/hour | Production, high-volume applications |

Rate limits can be customized per API key after creation.

### Compliance Frameworks

Specify compliance frameworks the customer needs:

- **HIPAA** - Healthcare applications handling PHI
- **SOC2** - Enterprise SaaS applications
- **GDPR** - European user data
- **PCI-DSS** - Payment card data

This information is stored for auditing but doesn't automatically enforce additional rules (that's future functionality).

## Error Handling

```python
import requests

def create_aptly_customer(email, company_name, plan):
    try:
        response = requests.post(
            "https://api.aptly.dev/v1/admin/customers",
            headers={
                "X-Admin-Secret": os.getenv("APTLY_ADMIN_SECRET"),
                "Content-Type": "application/json"
            },
            json={
                "email": email,
                "company_name": company_name,
                "plan": plan
            }
        )
        response.raise_for_status()
        return response.json()

    except requests.exceptions.HTTPError as e:
        if e.response.status_code == 409:
            print(f"Customer {email} already exists")
            return None
        elif e.response.status_code == 401:
            print("Invalid admin secret - check your environment variables")
            raise
        else:
            print(f"Error creating customer: {e.response.json()}")
            raise
```

## Automation Example

Script to bulk-create customers from a CSV:

```python
import csv
import requests
import os

ADMIN_SECRET = os.getenv("APTLY_ADMIN_SECRET")

def create_customer(email, company, plan):
    response = requests.post(
        "https://api.aptly.dev/v1/admin/customers",
        headers={"X-Admin-Secret": ADMIN_SECRET},
        json={
            "email": email,
            "company_name": company,
            "plan": plan
        }
    )
    response.raise_for_status()
    return response.json()

# Read customers from CSV
with open('customers.csv', 'r') as f:
    reader = csv.DictReader(f)
    for row in reader:
        try:
            result = create_customer(
                row['email'],
                row['company'],
                row['plan']
            )

            # Save API key securely
            with open('api_keys.txt', 'a') as keys_file:
                keys_file.write(
                    f"{row['email']},{result['api_key']['key']}\n"
                )

            print(f"✓ Created: {row['email']}")

        except Exception as e:
            print(f"✗ Failed: {row['email']} - {e}")
```
