title: "Health Check"
api: "GET /v1/health"
description: "Check the health status of the Aptly API and its dependencies"
---

The health endpoint provides a simple way to verify that the Aptly service is running and can connect to its required dependencies (database and Redis).

## Authentication

**None required** - This endpoint is publicly accessible.

## Response

Returns the current health status of the service and its dependencies.

<ResponseField name="status" type="string" required>
  Overall health status: `healthy` or `degraded`
</ResponseField>

<ResponseField name="version" type="string" required>
  API version string (e.g., "3.0.0")
</ResponseField>

<ResponseField name="checks" type="object" required>
  Individual health checks for dependencies

  <Expandable title="checks">
    <ResponseField name="database" type="string">
      Database connectivity: `ok` or `error`
    </ResponseField>

    <ResponseField name="redis" type="string">
      Redis connectivity: `ok` or `error`
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl https://api-aptly.nsquaredlabs.com/v1/health
```

```python Python
import requests

response = requests.get("https://api-aptly.nsquaredlabs.com/v1/health")
health = response.json()

if health["status"] == "healthy":
    print("✓ Service is healthy")
else:
    print(f"⚠ Service degraded: {health['checks']}")
```

```javascript JavaScript
const response = await fetch('https://api-aptly.nsquaredlabs.com/v1/health');
const health = await response.json();

console.log(`Status: ${health.status}`);
console.log(`Database: ${health.checks.database}`);
console.log(`Redis: ${health.checks.redis}`);
```

</CodeGroup>

## Use Cases

### Deployment Verification

Use the health endpoint to verify successful deployment:

```bash
#!/bin/bash
# scripts/verify_deployment.sh

HEALTH=$(curl -s https://your-app.railway.app/v1/health | jq -r '.status')

if [ "$HEALTH" = "healthy" ]; then
    echo "✓ Deployment successful"
    exit 0
else
    echo "✗ Deployment failed - service unhealthy"
    exit 1
fi
```

### Load Balancer Health Checks

Configure your load balancer to use this endpoint for health checks:

- **Path:** `/v1/health`
- **Expected Status:** 200 OK
- **Expected Response:** `"status": "healthy"`
- **Interval:** 30 seconds
- **Timeout:** 5 seconds

### Monitoring & Alerting

Set up monitoring alerts when status becomes `degraded`:

```python
import requests
import time

def monitor_health():
    while True:
        response = requests.get("https://api-aptly.nsquaredlabs.com/v1/health")
        health = response.json()

        if health["status"] != "healthy":
            send_alert(f"Aptly service degraded: {health['checks']}")

        time.sleep(60)  # Check every minute
```

## Notes

- The endpoint always returns HTTP 200, even when degraded
- Check the `status` field to determine actual health
- Individual component status is available in the `checks` object
- No authentication required makes it suitable for public monitoring
