The chat completions endpoint is the **main feature** of Aptly. It provides an OpenAI-compatible API that automatically redacts PII from requests before sending them to LLM providers, logs all requests for compliance, and enforces rate limits.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Request Body

<ParamField body="model" type="string" required>
  The model identifier (e.g., `gpt-4`, `claude-3-5-sonnet-20241022`, `gemini-pro`)
</ParamField>

<ParamField body="messages" type="array" required>
  Array of message objects with `role` and `content` fields

  ```json
  [
    {"role": "system", "content": "You are a helpful assistant"},
    {"role": "user", "content": "Hello!"}
  ]
  ```
</ParamField>

<ParamField body="api_keys" type="object" required>
  LLM provider API keys. Required key depends on the model:

  - `openai`: For GPT models (gpt-4, gpt-3.5-turbo, etc.)
  - `anthropic`: For Claude models
  - `google`: For Gemini models
  - `cohere`: For Cohere models
  - `together`: For Together AI models

  ```json
  {
    "openai": "sk-your-key-here"
  }
  ```

  **Note:** These keys are used for the request only and are never stored by Aptly.
</ParamField>

<ParamField body="stream" type="boolean" default={false}>
  Whether to stream the response using Server-Sent Events (SSE)
</ParamField>

<ParamField body="temperature" type="number">
  Sampling temperature between 0 and 2 (default varies by provider)
</ParamField>

<ParamField body="max_tokens" type="integer">
  Maximum tokens to generate in the response
</ParamField>

<ParamField body="top_p" type="number">
  Nucleus sampling parameter (0 to 1)
</ParamField>

<ParamField body="frequency_penalty" type="number">
  Penalty for token frequency (-2.0 to 2.0)
</ParamField>

<ParamField body="presence_penalty" type="number">
  Penalty for token presence (-2.0 to 2.0)
</ParamField>

<ParamField body="stop" type="string | array">
  Stop sequences where the API will stop generating
</ParamField>

<ParamField body="user" type="string">
  Unique identifier for the end-user (for usage tracking)
</ParamField>

<ParamField body="redact_response" type="boolean" default={false}>
  Whether to detect PII in the LLM's response and redact it
</ParamField>

### With PII Redaction

This example shows how Aptly automatically redacts sensitive information:

```python
import requests

response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={
        "Authorization": "Bearer apt_live_xyz789",
        "Content-Type": "application/json"
    },
    json={
        "model": "gpt-4",
        "messages": [
            {
                "role": "user",
                "content": "My name is John Smith and my SSN is 123-45-6789. Can you help me?"
            }
        ],
        "api_keys": {
            "openai": "sk-your-openai-key"
        }
    }
)

result = response.json()

# What the LLM actually received (PII redacted):
# "My name is PERSON_A and my SSN is SSN_A. Can you help me?"

print(f"PII detected: {result['pii_redacted']['entities_found']}")
# Output: PII detected: ['PERSON', 'US_SSN']

print(f"Redaction count: {result['pii_redacted']['redaction_count']}")
# Output: Redaction count: 2
```

### Streaming Response

```python
import requests

response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={
        "Authorization": "Bearer apt_live_xyz789",
        "Content-Type": "application/json"
    },
    json={
        "model": "gpt-4",
        "messages": [
            {"role": "user", "content": "Write a short poem"}
        ],
        "api_keys": {
            "openai": "sk-your-openai-key"
        },
        "stream": True
    },
    stream=True
)

# Process Server-Sent Events
for line in response.iter_lines():
    if line:
        line_text = line.decode('utf-8')
        if line_text.startswith('data: '):
            data = line_text[6:]  # Remove 'data: ' prefix
            if data != '[DONE]':
                import json
                chunk = json.loads(data)
                content = chunk['choices'][0]['delta'].get('content', '')
                if content:
                    print(content, end='', flush=True)
```

## How PII Redaction Works

When you send a request with PII:

1. **Detection**: Aptly scans the input messages for PII using Microsoft Presidio
2. **Redaction**: PII is replaced with tokens like `PERSON_A`, `EMAIL_A`, `SSN_A`
3. **LLM Call**: The redacted messages are sent to the LLM provider
4. **Response**: The LLM responds using the redacted tokens
5. **Audit Log**: Both original and redacted messages are logged (separately)

### Supported PII Entity Types

- `PERSON` - Person names
- `EMAIL_ADDRESS` - Email addresses
- `PHONE_NUMBER` - Phone numbers
- `US_SSN` - Social Security Numbers
- `CREDIT_CARD` - Credit card numbers
- `US_DRIVER_LICENSE` - Driver's license numbers
- `US_PASSPORT` - Passport numbers
- `IBAN_CODE` - International bank account numbers
- And more via [Presidio](https://microsoft.github.io/presidio/)

### Redaction Modes

You can configure the redaction mode in your customer profile (`PATCH /v1/me`):

- **mask** (default): `"John Smith"` → `"PERSON_A"`
- **hash**: `"John Smith"` → `"HASH_a3f2c1b9"`
- **remove**: `"John Smith"` → `"[REDACTED]"`

See the [PII Redaction Guide](/guides/pii-redaction) for details.

## Rate Limiting

Every request is subject to rate limiting based on your API key's configuration:

- Default: 100 requests per hour
- Rate limit headers are included in every response:
  - `X-RateLimit-Limit`: Maximum requests per hour
  - `X-RateLimit-Remaining`: Requests remaining
  - `X-RateLimit-Reset`: Unix timestamp when limit resets

When rate limited, you'll receive a `429` error with reset time.

## Audit Logging

Every request is automatically logged with:

- Timestamp and duration
- Model and token usage
- Cost calculation
- PII detections (input and response)
- Original and redacted messages (stored separately)
- User identifier (if provided)

Logs are immutable and queryable via the [Audit Logs API](/api/audit-logs/query-logs).

## Best Practices

### Security

1. **Never log provider API keys** - They're only used for requests
2. **Use environment-specific Aptly keys** - Separate `apt_test_*` and `apt_live_*`
3. **Rotate keys regularly** - Create new keys and revoke old ones

### Performance

1. **Use streaming for long responses** - Better UX for users
2. **Set appropriate `max_tokens`** - Control costs and latency
3. **Monitor rate limits** - Check response headers

### Compliance

1. **Enable `redact_response: true`** for sensitive use cases
2. **Review audit logs regularly** - Check for PII leakage
3. **Set appropriate compliance frameworks** in your profile

### Cost Management

1. **Track usage via audit logs** - Monitor costs per user/model
2. **Use cheaper models** when appropriate (e.g., `gpt-3.5-turbo`)
3. **Implement client-side rate limiting** - Avoid 429 errors

## Error Handling

Always handle these common errors:

```python
import requests

try:
    response = requests.post(
        "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
        headers={"Authorization": f"Bearer {api_key}"},
        json={...}
    )
    response.raise_for_status()
    result = response.json()

except requests.exceptions.HTTPError as e:
    if e.response.status_code == 429:
        # Rate limited - wait and retry
        reset_at = e.response.json()['error']['details']['reset_at']
        print(f"Rate limited. Retry after {reset_at}")
    elif e.response.status_code == 402:
        # LLM provider key invalid
        print("Provider API key rejected")
    elif e.response.status_code == 502:
        # LLM provider error - transient
        print("LLM provider error - retry")
    else:
        print(f"Error: {e.response.json()}")
```
