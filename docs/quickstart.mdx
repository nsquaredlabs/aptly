# Quickstart Guide

Get started with Aptly's compliance-as-a-service API in under 5 minutes. This guide shows you how to make your first PII-safe LLM request.

## Prerequisites

Before starting, you'll need:

## Step 1: Get Your API Key

Your Aptly account administrator will provide you with an API key that looks like:

```
apt_live_7a8b9c0d1e2f3g4h5i6j7k8l9m0n1o2p
```

<Warning>
  **Keep your API key secure!** Treat it like a password. Never commit it to version control.
</Warning>

Store it as an environment variable:

```bash
export APTLY_API_KEY="apt_live_7a8b9c0d1e2f3g4h5i6j7k8l9m0n1o2p"
```

## Step 3: Make Your First PII-Safe Request

Send a chat completion request with sensitive data. Aptly will automatically detect and redact PII before sending to your LLM provider:

<CodeGroup>
```bash cURL
curl -X POST https://api-aptly.nsquaredlabs.com/v1/chat/completions \
  -H "Authorization: Bearer $APTLY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {
        "role": "user",
        "content": "My email is john@example.com and my SSN is 123-45-6789. Can you help me with my account?"
      }
    ],
    "api_keys": {
      "openai": "sk-your-openai-key-here"
    }
  }'
```

```python Python
import requests
import os

response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={
        "Authorization": f"Bearer {os.getenv('APTLY_API_KEY')}",
        "Content-Type": "application/json"
    },
    json={
        "model": "gpt-4",
        "messages": [
            {
                "role": "user",
                "content": "My email is john@example.com and my SSN is 123-45-6789. Can you help me with my account?"
            }
        ],
        "api_keys": {
            "openai": os.getenv("OPENAI_API_KEY")
        }
    }
)

result = response.json()
print(f"Response: {result['choices'][0]['message']['content']}")
print(f"PII Detected: {result['pii_detections']}")
```

```javascript JavaScript
const response = await fetch('https://api-aptly.nsquaredlabs.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.APTLY_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'gpt-4',
    messages: [
      {
        role: 'user',
        content: 'My email is john@example.com and my SSN is 123-45-6789. Can you help me with my account?'
      }
    ],
    api_keys: {
      openai: process.env.OPENAI_API_KEY
    }
  })
});

const result = await response.json();
console.log('Response:', result.choices[0].message.content);
console.log('PII Detected:', result.pii_detections);
```
</CodeGroup>

<Tip>
  You provide your **own LLM provider API keys** in each request. Aptly never stores them.
</Tip>

<Check>

</Check>

## What Just Happened?

Here's what Aptly did behind the scenes:

1. **Authenticated** your request using your API key
2. **Scanned** the message for PII using Microsoft Presidio
3. **Detected** sensitive data: `john@example.com` and `123-45-6789`
4. **Redacted** them before sending to OpenAI:
   - `john@example.com` → `EMAIL_A`
   - `123-45-6789` → `SSN_A`
5. **Routed** the request to OpenAI using your provided API key
6. **Logged** the full transaction to an immutable audit log
7. **Returned** the response with PII detection metadata

## Step 5: View Your Audit Logs

Every request through Aptly is logged for compliance and auditing:

<CodeGroup>
```bash cURL
curl https://api-aptly.nsquaredlabs.com/v1/logs \
  -H "Authorization: Bearer $APTLY_API_KEY"
```

```python Python
import requests
import os

response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/logs",
    headers={"Authorization": f"Bearer {os.getenv('APTLY_API_KEY')}"}
)

logs = response.json()
for log in logs['logs']:
    print(f"{log['timestamp']}: {log['model']} - {log['status']}")
```

```javascript JavaScript
const response = await fetch('https://api-aptly.nsquaredlabs.com/v1/logs', {
  headers: {
    'Authorization': `Bearer ${process.env.APTLY_API_KEY}`
  }
});

const logs = await response.json();
logs.logs.forEach(log => {
  console.log(`${log.timestamp}: ${log.model} - ${log.status}`);
});
```
</CodeGroup>

Audit logs include:
- ✅ Original and redacted messages
- ✅ Token usage and costs
- ✅ PII detections with confidence scores
- ✅ Timestamps and model used
- ✅ Request/response payloads

## Common Use Cases

### Healthcare (HIPAA Compliance)

Redact patient names, medical record numbers, and dates:

```python
response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={"Authorization": f"Bearer {aptly_key}"},
    json={
        "model": "gpt-4",
        "messages": [{
            "role": "user",
            "content": "Patient Sarah Johnson (MRN: 12345) presented on 01/15/2026..."
        }],
        "api_keys": {"openai": openai_key},
        "pii_redaction_mode": "hash"  # Consistent redaction across requests
    }
)
```

### Financial Services

Protect credit card numbers and account numbers:

```python
response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={"Authorization": f"Bearer {aptly_key}"},
    json={
        "model": "gpt-4",
        "messages": [{
            "role": "user",
            "content": "Card ending in 4242, exp 12/25, issue with transaction..."
        }],
        "api_keys": {"openai": openai_key},
        "redact_response": True  # Also scan LLM responses for PII
    }
)
```

### Customer Support

Redact emails, phone numbers, and addresses:

```python
response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={"Authorization": f"Bearer {aptly_key}"},
    json={
        "model": "claude-3-5-sonnet-20241022",
        "messages": [{
            "role": "user",
            "content": "Customer john.doe@email.com called from (555) 123-4567..."
        }],
        "api_keys": {"anthropic": anthropic_key},
        "user": "support-agent-42"  # Track usage per agent
    }
)
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="401 Unauthorized">
    - Verify your API key is correct and starts with `apt_live_` or `apt_test_`
    - Ensure you're using the `Authorization: Bearer` header format
    - Check that your API key hasn't been revoked
  </Accordion>

  <Accordion title="402 Payment Required">
    - Your LLM provider API key was rejected
    - Verify the provider key is valid and has credits
    - Check that the key matches the model provider (e.g., OpenAI key for GPT-4)
  </Accordion>

  <Accordion title="429 Rate Limit Exceeded">
    - Your Aptly plan has rate limits (check response headers)
    - Contact support to increase limits or upgrade your plan
    - Implement exponential backoff in your client
  </Accordion>

  <Accordion title="PII not being detected">
    - Check the `pii_detections` field in the response
    - Some entity types require higher confidence scores
    - See the [PII Redaction Guide](/guides/pii-redaction) for supported types
  </Accordion>
</AccordionGroup>

Need help? Contact support@aptly.dev or check the [full troubleshooting guide](/troubleshooting).

## OpenAI Compatibility

Aptly is a drop-in replacement for OpenAI's API. Just change the base URL:

```python
# Before (direct to OpenAI)
import openai
client = openai.OpenAI(api_key="sk-...")
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Hello"}]
)

# After (through Aptly with PII protection)
import openai
client = openai.OpenAI(
    api_key="apt_live_...",  # Your Aptly key
    base_url="https://api-aptly.nsquaredlabs.com/v1"
)
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "My SSN is 123-45-6789"}],
    extra_body={"api_keys": {"openai": "sk-..."}}
)
```
