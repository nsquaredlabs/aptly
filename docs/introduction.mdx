# What is Aptly?

Aptly is a **compliance-as-a-service middleware** that sits between your application and LLM providers (OpenAI, Anthropic, Google, etc.), automatically handling PII redaction, audit logging, and compliance requirements.

## The Problem

Organizations want to use Large Language Models but face critical challenges:

<CardGroup cols={2}>
  <Card title="PII Leakage Risk" icon="shield-halved">
    Sending customer data to external LLM APIs exposes sensitive information like SSNs, emails, and credit cards
  </Card>
  <Card title="Compliance Requirements" icon="gavel">
    HIPAA, SOC 2, and other frameworks require immutable audit trails of all data processing
  </Card>
  <Card title="Multi-Provider Complexity" icon="layer-group">
    Each LLM provider has different APIs, making it hard to switch or use multiple providers
  </Card>
  <Card title="Security Overhead" icon="lock">
    Building PII detection, redaction, and audit systems from scratch takes months
  </Card>
</CardGroup>

## The Solution

Aptly solves these problems with a **drop-in OpenAI-compatible proxy** that:

### рҹ”’ Automatic PII Redaction

Detects and redacts sensitive data before sending to LLMs using Microsoft Presidio:
- **Person names** - "John Smith" вҶ’ "PERSON_A"
- **Social Security Numbers** - "123-45-6789" вҶ’ "SSN_A"
- **Email addresses** - "user@company.com" вҶ’ "EMAIL_A"
- **Credit cards, phone numbers, and more**

Three redaction modes: **mask**, **hash**, or **remove**

### рҹ“Ӣ Immutable Audit Logs

Every request is logged with:
- Full request/response payloads
- PII detections and redactions
- Token usage and costs
- Timestamps and metadata
- Database-enforced immutability (can't be modified or deleted)

### рҹ”„ Multi-Provider Support

Single unified API works with:
- OpenAI (GPT-4, GPT-3.5)
- Anthropic (Claude)
- Google (Gemini)
- Cohere
- Together AI
- Any provider supported by LiteLLM

### вҡЎ OpenAI-Compatible

Minimal code changes required:

```python
# Before (direct OpenAI)
import openai
response = openai.ChatCompletion.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Hello"}]
)

# After (via Aptly)
import requests
response = requests.post(
    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
    headers={"Authorization": "Bearer apt_live_..."},
    json={
        "model": "gpt-4",
        "messages": [{"role": "user", "content": "Hello"}],
        "api_keys": {"openai": "sk-..."}
    }
)
```

## Key Features

<AccordionGroup>
  <Accordion title="Customer-Controlled Keys">
    Customers provide their own LLM provider API keys in each request. Aptly never stores them, reducing liability and keeping billing under customer control.
  </Accordion>

  <Accordion title="Distributed Rate Limiting">
    Redis-based rate limiting that works across multiple workers with fail-open behavior (if Redis is down, requests still go through).
  </Accordion>

  <Accordion title="Streaming Support">
    Full support for Server-Sent Events (SSE) streaming responses with PII redaction on streamed chunks.
  </Accordion>

  <Accordion title="Response PII Detection">
    Optional detection of PII in LLM responses, allowing you to flag or redact sensitive data that models might generate.
  </Accordion>
</AccordionGroup>

## Use Cases

### Healthcare (HIPAA Compliance)
Redact patient names, medical record numbers, and health data before sending to LLMs for clinical documentation, summarization, or chatbots.

### Financial Services
Protect credit card numbers, SSNs, and account numbers while using LLMs for customer support, fraud detection, or document analysis.

### Enterprise SaaS
Enable employees to use LLMs for productivity without exposing customer PII, with full audit trails for security reviews.

### AI Product Development
Build AI features without worrying about PII leakage, knowing every request is logged and compliant.

## Tech Stack

- **FastAPI** - High-performance Python web framework
- **Supabase** - PostgreSQL database with built-in auth and real-time
- **Redis** - Distributed rate limiting and caching
- **LiteLLM** - Unified interface to 100+ LLM providers
- **Microsoft Presidio** - Enterprise-grade PII detection
- **Sentry** - Error tracking and monitoring
