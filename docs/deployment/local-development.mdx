This guide walks you through setting up Aptly on your local machine for development and testing. By the end, you'll have a fully functional development environment with hot-reloading.

## Prerequisites

Before you begin, ensure you have:

- **Python 3.11+** - Check with `python --version`
- **Git** - For cloning the repository
- **Supabase account** - Free tier works fine ([supabase.com](https://supabase.com))
- **Redis** - Local instance or cloud service

## Step 1: Clone Repository

```bash
git clone https://github.com/your-org/aptly.git
cd aptly
```

## Step 2: Install Dependencies

### Python Dependencies

```bash
# Install all requirements
pip install -r requirements.txt

# For development, also install dev dependencies
pip install -r requirements-dev.txt
```

**What gets installed:**
- **FastAPI** - Web framework
- **Uvicorn** - ASGI server
- **Supabase** - Database client
- **Redis** - Rate limiting
- **Presidio** - PII detection
- **spaCy** - NLP model
- **LiteLLM** - LLM provider abstraction
- **pytest** - Testing (dev only)
- **ruff** - Linting (dev only)

### Download spaCy Model

PII detection requires spaCy's English model:

```bash
python -m spacy download en_core_web_sm
```

This downloads ~40MB and takes ~30 seconds.

## Step 3: Set Up Supabase

### Create Project

1. Go to [supabase.com/dashboard](https://supabase.com/dashboard)
2. Click **New project**
3. Choose:
   - **Name**: aptly-dev
   - **Database Password**: Strong password (save it!)
   - **Region**: Closest to you
4. Wait for provisioning (~2 minutes)

### Run Migrations

Install Supabase CLI:

<CodeGroup>

```bash macOS
brew install supabase/tap/supabase
```

```bash Linux/WSL
npm install -g supabase
```

```bash Windows
scoop bucket add supabase https://github.com/supabase/scoop-bucket.git
scoop install supabase
```

</CodeGroup>

Link your project and push migrations:

```bash
# Link to your Supabase project
supabase link --project-ref your-project-ref

# Run migrations
supabase db push
```

<Note>
  Get your project ref from the Supabase dashboard URL: `https://supabase.com/dashboard/project/<project-ref>`
</Note>

### Get Credentials

1. Go to **Project Settings** → **API**
2. Copy:
   - **Project URL** → `SUPABASE_URL`
   - **service_role key** → `SUPABASE_SERVICE_KEY`

## Step 4: Set Up Redis

You need Redis for rate limiting. Choose one option:

### Option A: Local Redis (Recommended for Dev)

<CodeGroup>

```bash macOS
brew install redis
brew services start redis
```

```bash Linux/Ubuntu
sudo apt-get install redis-server
sudo systemctl start redis
```

```bash Docker
docker run -d -p 6379:6379 redis:latest
```

</CodeGroup>

**Test connection:**

```bash
redis-cli ping
# Should return: PONG
```

**Redis URL:** `redis://localhost:6379/0`

### Option B: Upstash (Cloud Redis)

For a cloud option with free tier:

1. Sign up at [upstash.com](https://upstash.com)
2. Create a new Redis database
3. Choose region closest to you
4. Copy the Redis URL

**Advantages:**
- No local installation needed
- Free tier: 10,000 commands/day
- Accessible from anywhere

## Step 5: Configure Environment

### Create .env File

```bash
cp .env.example .env
```

### Edit .env

Open `.env` and set the following:

<ParamField header="SUPABASE_URL" type="string" required>
  Your Supabase project URL (e.g., `https://abc123.supabase.co`)
</ParamField>

<ParamField header="SUPABASE_SERVICE_KEY" type="string" required>
  Your Supabase service role key (starts with `eyJ...`)
</ParamField>

<ParamField header="REDIS_URL" type="string" required>
  Redis connection string (default: `redis://localhost:6379/0`)
</ParamField>

<ParamField header="APTLY_ADMIN_SECRET" type="string" required>
  Any string for local development (e.g., `dev-secret-123`)
</ParamField>

<ParamField header="ENVIRONMENT" type="string">
  Set to `development`
</ParamField>

<ParamField header="LOG_LEVEL" type="string">
  Set to `debug` for verbose logging
</ParamField>

**Example .env:**

```bash
SUPABASE_URL=https://abcxyz.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
REDIS_URL=redis://localhost:6379/0
APTLY_ADMIN_SECRET=dev-secret-123
ENVIRONMENT=development
LOG_LEVEL=debug
```

<Warning>
  Never commit `.env` to version control! It's already in `.gitignore`.
</Warning>

## Step 6: Run the Server

Start the development server with hot-reloading:

```bash
uvicorn src.main:app --reload --port 8000
```

**Expected output:**

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process [12345] using StatReload
INFO:     Started server process [12346]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

**Server is ready!** Visit [http://localhost:8000/docs](http://localhost:8000/docs) for interactive API docs.

```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Authorization: Bearer apt_live_..." \
  -H "Content-Type: application/json" \
```

## Development Workflow

### Hot Reloading

Uvicorn's `--reload` flag automatically restarts the server when you edit code:

1. Edit a file in `src/`
2. Save the file
3. Server automatically reloads
4. Test your changes immediately

**Files watched:**
- `src/**/*.py`
- `.env` (requires manual restart)

### Running Tests

```bash
# Run all tests
pytest tests/ -v

# Run with coverage
pytest tests/ -v --cov=src --cov-report=term-missing

# Run a single test file
pytest tests/test_chat_completions.py -v

# Run a specific test
pytest tests/test_chat_completions.py::test_chat_completion_basic -v

# Run tests matching a pattern
pytest tests/ -v -k "pii"
```

**Expected:** All tests should pass in <5 seconds.

### Code Quality

```bash
# Check code style and errors
ruff check src/ tests/

# Auto-fix issues
ruff check src/ tests/ --fix

# Type checking
mypy src/

# Format code
black src/ tests/
```

### Interactive API Docs

Visit [http://localhost:8000/docs](http://localhost:8000/docs) for:
- Interactive API documentation
- Try out endpoints in browser
- See request/response schemas
- Test authentication

**Alternative docs:** [http://localhost:8000/redoc](http://localhost:8000/redoc)

### Testing PII Redaction

Create test messages with PII:

```python
import requests

response = requests.post(
    "http://localhost:8000/v1/chat/completions",
    headers={
        "Authorization": "Bearer apt_live_...",
        "Content-Type": "application/json"
    },
    json={
        "model": "gpt-3.5-turbo",
        "messages": [
            {
                "role": "user",
                "content": "My SSN is 123-45-6789 and email is john@example.com"
            }
        ],
        "api_keys": {"openai": "sk-..."}
    }
)

# Check audit logs to see PII detection
log_id = response.json()["id"]
log = requests.get(
    f"http://localhost:8000/v1/logs/{log_id}",
    headers={"Authorization": "Bearer apt_live_..."}
).json()

print("Original:", log["messages_original"])
print("Redacted:", log["messages_redacted"])
print("PII Detected:", log["pii_entities_input"])
```

## Common Issues

### Port Already in Use

**Error:** `Address already in use`

**Solution:**

```bash
# Find process using port 8000
lsof -i :8000

# Kill the process
kill -9 <PID>

# Or use a different port
uvicorn src.main:app --reload --port 8001
```

### spaCy Model Not Found

**Error:** `Can't find model 'en_core_web_sm'`

**Solution:**

```bash
# Download the model again
python -m spacy download en_core_web_sm

# Verify installation
python -c "import spacy; nlp = spacy.load('en_core_web_sm'); print('OK')"
```

### Database Connection Error

**Error:** `Error connecting to database`

**Solutions:**

- ✅ Verify `SUPABASE_URL` and `SUPABASE_SERVICE_KEY` are correct
- ✅ Check that you ran migrations: `supabase db push`
- ✅ Test connection: `psql $DATABASE_URL` (get URL from Supabase)
- ✅ Check Supabase project is not paused (free tier pauses after 7 days inactivity)

### Redis Connection Error

**Error:** `Error connecting to Redis`

**Solutions:**

- ✅ Check Redis is running: `redis-cli ping`
- ✅ Verify `REDIS_URL` is correct
- ✅ For local Redis: `brew services start redis` or `sudo systemctl start redis`
- ✅ For Docker: `docker ps` to check container is running

### Import Errors

**Error:** `ModuleNotFoundError: No module named 'src'`

**Solution:**

```bash
# Ensure you're in the project root
pwd  # Should show .../aptly

# Reinstall dependencies
pip install -r requirements.txt

# Run from project root
python -m src.main  # Instead of: python src/main.py
```

### Environment Variables Not Loading

**Error:** Settings not reflecting `.env` changes

**Solutions:**

- ✅ Restart the server (`.env` changes require restart)
- ✅ Check `.env` is in the project root
- ✅ Verify no syntax errors in `.env`
- ✅ Use `python-dotenv` for loading: Already configured in `src/config.py`
