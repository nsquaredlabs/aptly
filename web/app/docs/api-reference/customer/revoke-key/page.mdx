---
title: "Revoke API Key"
api: "DELETE /v1/api-keys/key_xyz789"
description: "Revoke an API key to permanently disable it"
---

Permanently revoke an API key to prevent it from being used for authentication. This action cannot be undone.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

<Warning>
  You **cannot revoke the API key you're currently using** to make this request. Use a different key to revoke others.
</Warning>

## Path Parameters

<ParamField path="key_id" type="string" required>
  The unique identifier of the key to revoke (e.g., `key_abc123`)

  You can get key IDs from the [List API Keys](/api/customer/list-keys) endpoint.
</ParamField>

## Response

Returns **204 No Content** on successful revocation (empty response body).

## Use Cases

### Revoke Compromised Key

Immediately revoke a key that has been compromised:

```python
import requests

def revoke_compromised_key(current_api_key, compromised_key_id):
    """
    Revoke a compromised key and create a replacement.

    IMPORTANT: Make sure current_api_key is NOT the compromised one!
    """
    # First, create a replacement key
    create_response = requests.post(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys",
        headers={"Authorization": "Bearer {current_api_key}"},
        json={"name": "Replacement for {compromised_key_id}"}
    )

    if create_response.status_code != 201:
        print("✗ Failed to create replacement key!")
        return False

    new_key = create_response.json()
    print("✓ Created replacement key: {new_key['key_prefix']}")
    print("  New Key: {new_key['key']}")
    print()

    # Now revoke the compromised key
    revoke_response = requests.delete(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys/{compromised_key_id}",
        headers={"Authorization": "Bearer {current_api_key}"}
    )

    if revoke_response.status_code == 204:
        print("✓ Compromised key {compromised_key_id} revoked")
        print()
        print("⚠ NEXT STEPS:")
        print("  1. Update affected applications with new key")
        print("  2. Monitor audit logs for unauthorized usage")
        print("  3. Investigate how the key was compromised")
        return True
    else:
        print("✗ Failed to revoke key: {revoke_response.json()}")
        return False

# Usage
revoke_compromised_key("apt_live_safe_key", "key_compromised123")
```

### Bulk Revoke Old Keys

Revoke all keys that haven't been used in a long time:

```python
import requests
from datetime import datetime, timedelta, timezone

def revoke_inactive_keys(current_api_key, days_inactive=90):
    """Revoke keys that haven't been used in X days."""
    # Get all keys
    list_response = requests.get(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys",
        headers={"Authorization": "Bearer {current_api_key}"}
    )
    all_keys = list_response.json()['api_keys']

    now = datetime.now(timezone.utc)
    cutoff_date = now - timedelta(days=days_inactive)

    inactive_keys = []
    for key in all_keys:
        # Skip the current key
        if key['key_prefix'] in current_api_key:
            continue

        last_used = key.get('last_used_at')
        if last_used is None:
            # Never used
            inactive_keys.append(key)
        else:
            last_used_dt = datetime.fromisoformat(last_used.replace('Z', '+00:00'))
            if last_used_dt < cutoff_date:
                inactive_keys.append(key)

    if not inactive_keys:
        print("✓ No inactive keys found (older than {days_inactive} days)")
        return

    print("Found {len(inactive_keys)} inactive keys:\n")
    for key in inactive_keys:
        print("  {key['name']} ({key['key_prefix']})")
        last_used = key.get('last_used_at', 'Never')
        print("    Last used: {last_used}\n")

    confirm = input("Revoke all {len(inactive_keys)} keys? (yes/no): ")
    if confirm.lower() != 'yes':
        print("Cancelled")
        return

    # Revoke each inactive key
    revoked_count = 0
    for key in inactive_keys:
        response = requests.delete(
            "https://api-aptly.nsquaredlabs.com/v1/api-keys/{key['id']}",
            headers={"Authorization": "Bearer {current_api_key}"}
        )

        if response.status_code == 204:
            print("✓ Revoked: {key['name']}")
            revoked_count += 1
        else:
            print("✗ Failed: {key['name']}")

    print("\n✓ Revoked {revoked_count}/{len(inactive_keys)} keys")

# Usage
revoke_inactive_keys("apt_live_xyz789", days_inactive=90)
```

### Revoke by Name Pattern

Revoke keys matching a specific pattern:

```python
def revoke_keys_by_pattern(current_api_key, pattern):
    """Revoke all keys whose names contain a specific pattern."""
    # Get all keys
    response = requests.get(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys",
        headers={"Authorization": "Bearer {current_api_key}"}
    )
    all_keys = response.json()['api_keys']

    # Find matching keys
    matching_keys = [
        key for key in all_keys
        if pattern.lower() in key['name'].lower()
        and key['key_prefix'] not in current_api_key
    ]

    if not matching_keys:
        print("No keys found matching '{pattern}'")
        return

    print("Found {len(matching_keys)} keys matching '{pattern}':\n")
    for key in matching_keys:
        print("  {key['name']} ({key['id']})")

    confirm = input("\nRevoke these {len(matching_keys)} keys? (yes/no): ")
    if confirm.lower() != 'yes':
        print("Cancelled")
        return

    # Revoke each
    for key in matching_keys:
        response = requests.delete(
            "https://api-aptly.nsquaredlabs.com/v1/api-keys/{key['id']}",
            headers={"Authorization": "Bearer {current_api_key}"}
        )

        if response.status_code == 204:
            print("✓ Revoked: {key['name']}")
        else:
            print("✗ Failed: {key['name']}")

# Usage - revoke all "test" or "development" keys
revoke_keys_by_pattern("apt_live_xyz789", "test")
```

### Safe Revocation with Verification

Revoke a key with safety checks:

```python
def safe_revoke_key(current_api_key, key_id_to_revoke):
    """Safely revoke a key with verification steps."""
    # Step 1: Get the key details
    list_response = requests.get(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys",
        headers={"Authorization": "Bearer {current_api_key}"}
    )
    all_keys = list_response.json()['api_keys']

    key_to_revoke = None
    for key in all_keys:
        if key['id'] == key_id_to_revoke:
            key_to_revoke = key
            break

    if not key_to_revoke:
        print("✗ Key {key_id_to_revoke} not found")
        return False

    # Step 2: Verify it's not the current key
    if key_to_revoke['key_prefix'] in current_api_key:
        print("✗ ERROR: Cannot revoke the current API key!")
        print("  Use a different key to revoke this one")
        return False

    # Step 3: Show key details and confirm
    print("Key to revoke:")
    print("  Name: {key_to_revoke['name']}")
    print("  ID: {key_to_revoke['id']}")
    print("  Prefix: {key_to_revoke['key_prefix']}")
    print("  Created: {key_to_revoke['created_at']}")
    print("  Last Used: {key_to_revoke.get('last_used_at', 'Never')}")
    print()

    confirm = input("Confirm revocation (type 'REVOKE' to proceed): ")
    if confirm != 'REVOKE':
        print("Cancelled")
        return False

    # Step 4: Revoke
    response = requests.delete(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys/{key_id_to_revoke}",
        headers={"Authorization": "Bearer {current_api_key}"}
    )

    if response.status_code == 204:
        print("\n✓ Key successfully revoked")
        print("⚠ This action cannot be undone")
        return True
    else:
        print("\n✗ Revocation failed: {response.json()}")
        return False

# Usage
safe_revoke_key("apt_live_xyz789", "key_abc123")
```

## Important Notes

### Permanent Action

<Warning>
  Revocation is **permanent and cannot be undone**. The revoked key will:

  - Immediately stop working for authentication
  - No longer appear in the [List API Keys](/api/customer/list-keys) endpoint
  - Remain visible to admins (via [Get Customer](/api/admin/get-customer))
  - Be recorded in audit logs for compliance
</Warning>

### Cannot Revoke Current Key

You **cannot revoke the API key you're currently using** to make the request. This prevents you from accidentally locking yourself out.

**Workflow for revoking your only key:**
1. Create a new API key using your current key
2. Verify the new key works
3. Use the new key to revoke the old key

### Revoked Keys in Audit Logs

Revoked keys remain in the audit logs history. You can query logs to see:
- When the key was used
- What requests were made
- When it was revoked

## Security Best Practices

1. **Revoke immediately when compromised** - Don't wait
2. **Create replacement first** - Don't revoke your only key
3. **Monitor audit logs** - Check for unauthorized usage before revoking
4. **Regular audits** - Review and revoke unused keys monthly
5. **Document revocations** - Keep records of why keys were revoked

## Common Errors

### Error: Cannot revoke current key

```json
{
  "error": {
    "type": "conflict",
    "message": "Cannot revoke the API key you're currently using"
  }
}
```

**Solution:** Use a different API key to revoke this one. If this is your only key, create a new one first.

### Error: Key not found

```json
{
  "error": {
    "type": "not_found",
    "message": "API key not found"
  }
}
```

**Possible reasons:**
- Key ID is incorrect
- Key has already been revoked
- Key belongs to a different customer
- You don't have permission to revoke this key
