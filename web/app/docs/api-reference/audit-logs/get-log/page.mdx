---
title: "Get Log Detail"
api: "GET /v1/logs/log_xyz789"
description: "Retrieve complete details of a specific audit log entry including messages and PII detections"
---

Get comprehensive details for a single audit log entry, including the original and redacted messages, PII detections, and full request/response metadata.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Path Parameters

<ParamField path="log_id" type="string" required>
  The unique audit log identifier (e.g., `log_abc123`)

  You can get log IDs from the [Query Logs](/api/audit-logs/query-logs) endpoint.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique log entry identifier
</ResponseField>

<ResponseField name="customer_id" type="string" required>
  Your customer identifier
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp of the request
</ResponseField>

<ResponseField name="model" type="string" required>
  LLM model used (e.g., `gpt-4`, `claude-3-5-sonnet-20241022`)
</ResponseField>

<ResponseField name="provider" type="string" required>
  LLM provider (e.g., `openai`, `anthropic`, `google`)
</ResponseField>

<ResponseField name="user_id" type="string">
  User identifier (if provided in request)
</ResponseField>

<ResponseField name="messages_original" type="array" required>
  Original messages before PII redaction

  <Warning>
    Contains unredacted PII. Handle with care for compliance.
  </Warning>
</ResponseField>

<ResponseField name="messages_redacted" type="array" required>
  Messages after PII redaction (sent to LLM)
</ResponseField>

<ResponseField name="response_content" type="string" required>
  The LLM's response content
</ResponseField>

<ResponseField name="pii_entities_input" type="array" required>
  List of PII entities detected in input

  <Expandable title="pii_entity">
    <ResponseField name="entity_type" type="string">
      Type of PII (e.g., `PERSON`, `EMAIL_ADDRESS`, `US_SSN`)
    </ResponseField>

    <ResponseField name="start" type="integer">
      Character position where entity starts
    </ResponseField>

    <ResponseField name="end" type="integer">
      Character position where entity ends
    </ResponseField>

    <ResponseField name="score" type="number">
      Confidence score (0.0 to 1.0)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pii_entities_response" type="array" required>
  List of PII entities detected in response (if `redact_response: true` was used)
</ResponseField>

<ResponseField name="tokens_input" type="integer" required>
  Input tokens used
</ResponseField>

<ResponseField name="tokens_output" type="integer" required>
  Output tokens generated
</ResponseField>

<ResponseField name="cost_usd" type="number" required>
  Estimated cost in USD
</ResponseField>

<ResponseField name="duration_ms" type="integer" required>
  Request duration in milliseconds
</ResponseField>

<ResponseField name="status" type="string" required>
  Request status: `success` or `error`
</ResponseField>

<ResponseField name="error_message" type="string">
  Error message if status is `error`
</ResponseField>

<ResponseField name="compliance_framework" type="string">
  Compliance framework associated with this request (e.g., `HIPAA`)
</ResponseField>

## Use Cases

### PII Detection Audit

Review what PII was detected and how it was redacted:

```python
import requests

def audit_pii_redaction(api_key, log_id):
    """Review PII redaction for a specific request."""
    response = requests.get(
        "https://api-aptly.nsquaredlabs.com/v1/logs/log_xyz789",
        headers={"Authorization": "Bearer {api_key}"}
    )
    log = response.json()

    print("=" * 60)
    print("PII REDACTION AUDIT")
    print("=" * 60)
    print("\nLog ID: {log['id']}")
    print("Timestamp: {log['timestamp']}")
    print("Model: {log['model']}")

    # Show original vs redacted
    original_msg = log['messages_original'][0]['content']
    redacted_msg = log['messages_redacted'][0]['content']

    print("\n--- ORIGINAL MESSAGE ---")
    print(original_msg)

    print("\n--- REDACTED MESSAGE (sent to LLM) ---")
    print(redacted_msg)

    # Show detected entities
    print("\n--- PII ENTITIES DETECTED ({len(log['pii_entities_input'])}) ---")
    for entity in log['pii_entities_input']:
        # Extract the actual text
        text = original_msg[entity['start']:entity['end']]
        print("\n{entity['entity_type']}:")
        print("  Text: '{text}'")
        print("  Position: {entity['start']}-{entity['end']}")
        print("  Confidence: {entity['score']:.2%}")

    # Response PII
    if log['pii_entities_response']:
        print("\n⚠ PII DETECTED IN RESPONSE:")
        for entity in log['pii_entities_response']:
            print("  - {entity['entity_type']} (score: {entity['score']:.2%})")

    print("=" * 60)

# Usage
audit_pii_redaction("apt_live_xyz789", "log_abc123")
```

### Extract PII Statistics

Analyze PII detection patterns:

```python
def extract_pii_statistics(api_key, log_id):
    """Extract statistics about PII detection."""
    response = requests.get(
        "https://api-aptly.nsquaredlabs.com/v1/logs/log_xyz789",
        headers={"Authorization": "Bearer {api_key}"}
    )
    log = response.json()

    from collections import Counter

    # Count entity types
    entity_types = [e['entity_type'] for e in log['pii_entities_input']]
    type_counts = Counter(entity_types)

    # Calculate average confidence
    avg_confidence = sum(e['score'] for e in log['pii_entities_input']) / len(log['pii_entities_input']) if log['pii_entities_input'] else 0

    print("PII Detection Statistics:")
    print("  Total entities: {len(log['pii_entities_input'])}")
    print("  Unique types: {len(type_counts)}")
    print("  Average confidence: {avg_confidence:.2%}")
    print("\nBreakdown:")
    for entity_type, count in type_counts.most_common():
        print("  {entity_type}: {count}")

    return {
        "total": len(log['pii_entities_input']),
        "types": dict(type_counts),
        "avg_confidence": avg_confidence
    }

# Usage
stats = extract_pii_statistics("apt_live_xyz789", "log_abc123")
```

## Important Notes

### PII Sensitivity

<Warning>
  The `messages_original` field contains **unredacted PII**. Handle this data carefully:

  - Only access when necessary for compliance audits
  - Never log or display in public interfaces
  - Ensure proper access controls
  - Follow your compliance framework's requirements
</Warning>

### Immutability

Audit logs are **immutable** and cannot be modified or deleted. This is enforced at the database level for compliance.

### Log Retention

Logs are retained according to your account's `retention_days` setting (default: ~7 years).

### Cost Estimates

The `cost_usd` field is an estimate based on known provider pricing and may not reflect exact costs.

## Understanding PII Entities

Common entity types you'll see:

| Entity Type | Description | Example |
|-------------|-------------|---------|
| `PERSON` | Person names | "John Smith" |
| `EMAIL_ADDRESS` | Email addresses | "john@example.com" |
| `PHONE_NUMBER` | Phone numbers | "+1-555-0100" |
| `US_SSN` | Social Security Numbers | "123-45-6789" |
| `CREDIT_CARD` | Credit card numbers | "4111-1111-1111-1111" |
| `US_DRIVER_LICENSE` | Driver's license | "D1234567" |
| `IBAN_CODE` | Bank account numbers | "GB29 NWBK..." |

See the [PII Redaction Guide](/guides/pii-redaction) for the complete list.
