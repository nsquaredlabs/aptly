# Export Analytics

**`GET /v1/analytics/export`**

Download analytics data as a CSV or JSON file

Download a daily-aggregated summary of your analytics as a CSV or JSON file. Each row represents one calendar day. Designed for compliance reporting and offline analysis.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Query Parameters

<ParamField query="start_date" type="string" required>
  Start of the export period.

  Formats accepted:
  - ISO 8601: `2026-01-01T00:00:00Z`
  - Date only: `2026-01-01`
</ParamField>

<ParamField query="end_date" type="string" required>
  End of the export period.

  Formats accepted:
  - ISO 8601: `2026-01-31T23:59:59Z`
  - Date only: `2026-01-31`
</ParamField>

<ParamField query="format" type="string" default="csv">
  File format for the download.

  Accepted values: `csv`, `json`
</ParamField>

<ParamField query="include" type="string">
  Comma-separated list of data sections to include: `usage`, `models`, `users`, `pii`.

  Default: all sections
</ParamField>

## Response

The response is a file download. The browser receives a `Content-Disposition` header with the suggested filename (e.g., `aptly-analytics-2026-01.csv`).

### CSV Columns

| Column | Type | Description |
|---|---|---|
| `date` | string | Date (`YYYY-MM-DD`) |
| `requests` | integer | Total requests on this day |
| `tokens_input` | integer | Input tokens consumed |
| `tokens_output` | integer | Output tokens generated |
| `cost_usd` | number | Estimated cost in USD |
| `avg_latency_ms` | integer | Average latency in milliseconds |
| `pii_detected_count` | integer | Requests with PII in input or response |
| `top_model` | string | Most-used model on this day |

### JSON

When `format=json` the body is a JSON array using the same fields as the CSV columns above.

## Example Requests

<CodeGroup>

```bash cURL
curl -o analytics.csv \
  "https://api-aptly.nsquaredlabs.com/v1/analytics/export?start_date=2026-01-01&end_date=2026-01-31&format=csv" \
  -H "Authorization: Bearer apt_live_xyz789"
```

```python Python
import requests

# Download CSV
response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/analytics/export",
    headers={"Authorization": "Bearer apt_live_xyz789"},
    params={
        "start_date": "2026-01-01",
        "end_date": "2026-01-31",
        "format": "csv"
    }
)

with open("aptly-analytics-january.csv", "w") as f:
    f.write(response.text)

print("Saved analytics CSV")

# Or download JSON and process directly
response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/analytics/export",
    headers={"Authorization": "Bearer apt_live_xyz789"},
    params={
        "start_date": "2026-01-01",
        "end_date": "2026-01-31",
        "format": "json"
    }
)

for day in response.json():
    print("{day['date']}: {day['requests']} requests, ${day['cost_usd']:.2f}")
```

```javascript JavaScript
const response = await fetch(

const csv = await response.text();
```

</CodeGroup>

### Filter Specific Sections

```python
# Export only usage and PII sections
response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/analytics/export",
    headers={"Authorization": "Bearer apt_live_xyz789"},
    params={
        "start_date": "2026-01-01",
        "end_date": "2026-01-31",
        "format": "csv",
        "include": "usage,pii"
    }
)
```

## Notes

- Both `start_date` and `end_date` are required. The endpoint returns `422` if either is missing.
- An empty period returns a CSV with only the header row, or an empty JSON array `[]`.
- `top_model` is the single most-used model on that day. Ties are broken arbitrarily.
- `cost_usd` is an estimate based on token counts and provider pricing.
