# Model Breakdown

**`GET /v1/analytics/models`**

Get usage and cost breakdown by model and provider

See which models and providers your team is using most, along with token and cost metrics for each. Results are sorted by request count descending.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Query Parameters

<ParamField query="start_date" type="string">
  Start of the analysis period.

  Formats accepted:
  - ISO 8601: `2026-01-01T00:00:00Z`
  - Date only: `2026-01-01`

  Default: 30 days ago
</ParamField>

<ParamField query="end_date" type="string">
  End of the analysis period.

  Formats accepted:
  - ISO 8601: `2026-01-31T23:59:59Z`
  - Date only: `2026-01-31`

  Default: Current time
</ParamField>

## Response

<ResponseField name="models" type="array" required>
  Per-model usage breakdown, sorted by request count (highest first)

  <Expandable title="model entry">
    <ResponseField name="model" type="string">
      Model identifier (e.g., `gpt-4`, `claude-3-sonnet`)
    </ResponseField>

    <ResponseField name="provider" type="string">
      Provider name (e.g., `openai`, `anthropic`)
    </ResponseField>

    <ResponseField name="requests" type="integer">
      Number of requests made to this model
    </ResponseField>

    <ResponseField name="tokens_input" type="integer">
      Total input tokens sent to this model
    </ResponseField>

    <ResponseField name="tokens_output" type="integer">
      Total output tokens generated by this model
    </ResponseField>

    <ResponseField name="cost_usd" type="number">
      Estimated total cost for this model in USD
    </ResponseField>

    <ResponseField name="avg_latency_ms" type="integer">
      Average response latency for this model in milliseconds
    </ResponseField>

    <ResponseField name="percentage_of_requests" type="number">
      This model's share of total requests as a percentage
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="period_start" type="string" required>
  Effective start date of the period (`YYYY-MM-DD`)
</ResponseField>

<ResponseField name="period_end" type="string" required>
  Effective end date of the period (`YYYY-MM-DD`)
</ResponseField>

## Example Requests

<CodeGroup>

```bash cURL
curl "https://api-aptly.nsquaredlabs.com/v1/analytics/models?start_date=2026-01-01&end_date=2026-01-31" \
  -H "Authorization: Bearer apt_live_xyz789"
```

```python Python
import requests

response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/analytics/models",
    headers={"Authorization": "Bearer apt_live_xyz789"},
    params={
        "start_date": "2026-01-01",
        "end_date": "2026-01-31"
    }
)

data = response.json()
for model in data["models"]:
    print("{model['model']} ({model['provider']})")
    print("  {model['requests']} requests ({model['percentage_of_requests']}%)")
    print("  Cost: ${model['cost_usd']:.2f} | Avg latency: {model['avg_latency_ms']}ms\n")
```

```javascript JavaScript
const response = await fetch(
  'https://api-aptly.nsquaredlabs.com/v1/analytics/models?' +
  new URLSearchParams({ start_date: '2026-01-01', end_date: '2026-01-31' }),
  { headers: { 'Authorization': 'Bearer apt_live_xyz789' } }
);

const data = await response.json();
data.models.forEach(m => {
  console.log(`${m.model}: ${m.requests} requests (${m.percentage_of_requests}%), $${m.cost_usd}`);
});
```

</CodeGroup>

## Notes

- `percentage_of_requests` values across all models sum to 100%.
- A model + provider pair is a single entry. If you route the same model name through different providers each combination appears separately.
- An empty `models` array is returned when there are no requests in the period.
