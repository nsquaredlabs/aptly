# Usage Summary

**`GET /v1/analytics/usage`**

Get aggregated usage metrics with time series data for a given period

Retrieve a summary of your API usage over time, broken down into configurable time buckets. Returns both aggregate totals and a time series suitable for charting.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Query Parameters

<ParamField query="start_date" type="string">
  Start of the analysis period.

  Formats accepted:
  - ISO 8601: `2026-01-01T00:00:00Z`
  - Date only: `2026-01-01`

  Default: 30 days ago
</ParamField>

<ParamField query="end_date" type="string">
  End of the analysis period.

  Formats accepted:
  - ISO 8601: `2026-01-31T23:59:59Z`
  - Date only: `2026-01-31`

  Default: Current time
</ParamField>

<ParamField query="granularity" type="string" default="day">
  Time bucket size for the `time_series` array.

  Accepted values: `day`, `week`, `month`
</ParamField>

## Response

<ResponseField name="summary" type="object" required>
  Aggregate totals for the requested period

  <Expandable title="summary">
    <ResponseField name="total_requests" type="integer">
      Total number of API requests in the period
    </ResponseField>

    <ResponseField name="total_tokens" type="integer">
      Total tokens consumed (input + output combined)
    </ResponseField>

    <ResponseField name="total_cost_usd" type="number">
      Total estimated cost in USD
    </ResponseField>

    <ResponseField name="avg_latency_ms" type="integer">
      Average request latency in milliseconds
    </ResponseField>

    <ResponseField name="period_start" type="string">
      Effective start date of the period (`YYYY-MM-DD`)
    </ResponseField>

    <ResponseField name="period_end" type="string">
      Effective end date of the period (`YYYY-MM-DD`)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="time_series" type="array" required>
  Per-bucket usage breakdown, ordered chronologically

  <Expandable title="time series entry">
    <ResponseField name="date" type="string">
      Bucket start date (`YYYY-MM-DD`). For `week` granularity this is the Monday; for `month` it is the 1st.
    </ResponseField>

    <ResponseField name="requests" type="integer">
      Requests in this bucket
    </ResponseField>

    <ResponseField name="tokens_input" type="integer">
      Input tokens in this bucket
    </ResponseField>

    <ResponseField name="tokens_output" type="integer">
      Output tokens in this bucket
    </ResponseField>

    <ResponseField name="cost_usd" type="number">
      Estimated cost for this bucket in USD
    </ResponseField>

    <ResponseField name="avg_latency_ms" type="integer">
      Average latency for requests in this bucket in milliseconds
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Requests

<CodeGroup>

```bash cURL
curl "https://api-aptly.nsquaredlabs.com/v1/analytics/usage?start_date=2026-01-01&end_date=2026-01-31&granularity=day" \
  -H "Authorization: Bearer apt_live_xyz789"
```

```python Python
import requests

response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/analytics/usage",
    headers={"Authorization": "Bearer apt_live_xyz789"},
    params={
        "start_date": "2026-01-01",
        "end_date": "2026-01-31",
        "granularity": "week"
    }
)

data = response.json()
print("Total requests: {data['summary']['total_requests']}")
print("Total cost:     ${data['summary']['total_cost_usd']:.2f}\n")

for point in data["time_series"]:
    print("  {point['date']}: {point['requests']} requests, ${point['cost_usd']:.2f}")
```

```javascript JavaScript
const response = await fetch(
  'https://api-aptly.nsquaredlabs.com/v1/analytics/usage?' +
  new URLSearchParams({
    start_date: '2026-01-01',
    end_date: '2026-01-31',
    granularity: 'week'
  }),
  { headers: { 'Authorization': 'Bearer apt_live_xyz789' } }
);

const data = await response.json();
console.log(`Total requests: ${data.summary.total_requests}`);
data.time_series.forEach(point => {
  console.log(`  ${point.date}: ${point.requests} requests`);
});
```

</CodeGroup>

## Notes

- The `summary` totals always equal the sum of the corresponding `time_series` fields.
- With `week` granularity each `date` is the Monday of that week. With `month` it is the 1st of the month.
- `cost_usd` is an estimate based on token counts and provider pricing tables. Actual provider bills may differ.
- When no date parameters are provided the default range is the last 30 days.
