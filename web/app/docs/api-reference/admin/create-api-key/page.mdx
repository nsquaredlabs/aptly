# Create API Key for Customer

**`POST /v1/admin/customers/cus_abc123/api-keys`**

Generate an additional API key for an existing customer

Create a new API key for a customer with custom name and rate limit. This is useful when customers need multiple keys for different environments or teams.

## Authentication

Admin secret required via `X-Admin-Secret` header.

## Path Parameters

- `customer_id` (string, required) - The unique customer identifier (e.g., `cus_abc123`)

## Request Body

- `name` (string, required) - Human-readable name for the key (e.g., "Production Key", "Staging Environment")
- `rate_limit_per_hour` (integer, required) - Requests per hour allowed for this key

Common values:
- **100**: Free tier / development
- **1000**: Pro tier / production
- **5000**: Enterprise tier

## Response

Returns the created API key with the following fields:

- `id` (string) - Unique key identifier
- `key` (string) - **The full API key** (e.g., `apt_live_xyz789...`) - This is the only time the full key is returned
- `key_prefix` (string) - First 12 characters of the key (for identification)
- `name` (string) - The name provided for this key
- `rate_limit_per_hour` (integer) - Requests per hour allowed
- `created_at` (string) - ISO 8601 timestamp of key creation

## Example Request

```bash
curl -X POST "https://api-aptly.nsquaredlabs.com/v1/admin/customers/cus_abc123/api-keys" \
  -H "X-Admin-Secret: your-admin-secret" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production API Key",
    "rate_limit_per_hour": 1000
  }'
```

## Example Response

```json
{
  "id": "key_xyz789",
  "key": "apt_live_abc123def456ghi789jkl012mno345pqr678",
  "key_prefix": "apt_live_abc",
  "name": "Production API Key",
  "rate_limit_per_hour": 1000,
  "created_at": "2026-01-31T12:00:00Z"
}
```

## Error Responses

**404 - Customer Not Found**
```json
{
  "error": {
    "type": "not_found",
    "message": "Customer not found",
    "code": "NOT_FOUND"
  }
}
```

**401 - Unauthorized**
```json
{
  "detail": "Invalid admin secret"
}
```

## Rate Limit Guidelines

Choose appropriate rate limits based on use case:

- **Development**: 100/hour - Testing and development environments
- **Staging**: 500/hour - Pre-production testing
- **Production (Small)**: 1,000/hour - Small-scale production apps
- **Production (Medium)**: 5,000/hour - Medium-scale production apps
- **Production (Large)**: 10,000/hour - High-volume production apps
- **Enterprise**: Custom - Contact support for higher limits

**Note:** Rate limits are per API key, not per customer. A customer with 3 keys can make 3x the requests.

## Security Best Practices

### Immediate Storage

The full API key is only returned **once**. You must capture and store it immediately.

### Key Distribution

Never send API keys via:
- Email (not encrypted)
- Slack/Discord (logged forever)
- SMS (insecure)

Instead, use:
- Password managers (1Password, LastPass)
- Secrets managers (HashiCorp Vault, AWS Secrets Manager)
- Encrypted messaging (Signal, with disappearing messages)

### Naming Conventions

Use descriptive names that help identify keys:

**Good:**
- "Production - Web App"
- "Staging - API v2"
- "Team Backend - Microservices"

**Bad:**
- "Key 1"
- "Test"
- "asd"
