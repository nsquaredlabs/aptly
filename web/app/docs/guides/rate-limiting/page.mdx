# Rate Limiting Guide

Understanding and managing rate limits in Aptly

Aptly implements distributed rate limiting to prevent abuse and ensure fair usage. This guide explains how rate limits work and how to handle them in your application.

## How Rate Limiting Works

Every API key has a **requests per hour** limit. Rate limits are:

- **Per API key** - Each key has its own limit
- **Distributed** - Enforced via Redis (works across multiple servers)
- **Hourly reset** - Resets every hour (e.g., 10:00, 11:00, 12:00)
- **Fail-open** - If Redis is unavailable, requests are allowed (reliability over strict limiting)

## Rate Limit Tiers

| Plan | Default Limit | Typical Use Case |
|------|---------------|------------------|
| **Free** | 100 req/hour | Development, testing, small projects |
| **Pro** | 1,000 req/hour | Production applications |
| **Enterprise** | Custom | High-volume production |

## Rate Limit Headers

Every response includes rate limit headers:

```http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 847
X-RateLimit-Reset: 1706788800
```

| Header | Description | Example |
|--------|-------------|---------|
| `X-RateLimit-Limit` | Maximum requests per hour | `1000` |
| `X-RateLimit-Remaining` | Requests remaining this hour | `847` |
| `X-RateLimit-Reset` | Unix timestamp when limit resets | `1706788800` |

## Handling Rate Limits

### 429 Rate Limited Response

When you exceed your rate limit, you'll receive a `429` status code:

```json
{
  "error": {
    "type": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Limit: 1000 requests per hour.",
    "code": "RATE_LIMIT_EXCEEDED",
    "details": {
      "limit": 1000,
      "current": 1001,
      "reset_at": "2026-01-31T11:00:00Z"
    }
  }
}
```

### Retry Strategy

Implement exponential backoff with retry:

```python
import requests
import time
from datetime import datetime

def make_request_with_retry(api_key, payload, max_retries=3):
    """Make request with automatic retry on rate limit."""
    for attempt in range(max_retries):
        try:
            response = requests.post(
                "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
                headers={"Authorization": f"Bearer {api_key}"},
                json=payload
            )

            if response.status_code == 429:
                # Rate limited
                error = response.json()
                reset_at = error['error']['details']['reset_at']
                reset_time = datetime.fromisoformat(reset_at.replace('Z', '+00:00'))
                now = datetime.now(timezone.utc)
                wait_seconds = (reset_time - now).total_seconds()

                if wait_seconds > 0:
                    print(f"Rate limited. Waiting {wait_seconds:.0f}s until reset...")
                    time.sleep(wait_seconds + 1)  # Add 1s buffer
                    continue
                else:
                    # Reset time passed, retry immediately
                    continue

            # Success or other error
            response.raise_for_status()
            return response.json()

        except requests.exceptions.RequestException as e:
            if attempt == max_retries - 1:
                raise
            wait = 2 ** attempt  # Exponential backoff
            print(f"Request failed, retrying in {wait}s...")
            time.sleep(wait)

    raise Exception("Max retries exceeded")

# Usage
result = make_request_with_retry(
    "apt_live_xyz789",
    {
        "model": "gpt-4",
        "messages": [{"role": "user", "content": "Hello"}],
        "api_keys": {"openai": "sk-..."}
    }
)
```

### Check Before Sending

Avoid rate limits by checking beforehand:

```python
def should_make_request(api_key, threshold=0.9):
    """
    Check if we should make a request based on current usage.

    Args:
        api_key: Your Aptly API key
        threshold: Stop at this percentage (0.9 = 90%)

    Returns:
        bool: True if safe to make request
    """
    response = requests.get(
        "https://api-aptly.nsquaredlabs.com/v1/me",
        headers={"Authorization": f"Bearer {api_key}"}
    )
    usage = response.json()['usage']

    limit = usage['rate_limit_per_hour']
    current = usage['requests_this_hour']
    percent = current / limit

    if percent >= threshold:
        print(f"⚠ At {percent:.0%} of rate limit. Pausing...")
        return False

    return True

# Usage
if should_make_request("apt_live_xyz789"):
    # Make request
    response = requests.post(...)
else:
    # Wait or queue for later
    print("Rate limit threshold reached, waiting...")
```

## Multiple API Keys Strategy

Distribute load across multiple keys:

```python
class LoadBalancedClient:
    """Distribute requests across multiple API keys."""

    def __init__(self, api_keys):
        self.api_keys = api_keys
        self.current_index = 0

    def get_next_key(self):
        """Round-robin key selection."""
        key = self.api_keys[self.current_index]
        self.current_index = (self.current_index + 1) % len(self.api_keys)
        return key

    def make_request(self, payload):
        """Make request with automatic key rotation."""
        for attempt in range(len(self.api_keys)):
            api_key = self.get_next_key()

            response = requests.post(
                "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
                headers={"Authorization": f"Bearer {api_key}"},
                json=payload
            )

            if response.status_code == 429:
                # Try next key
                continue

            response.raise_for_status()
            return response.json()

        raise Exception("All API keys rate limited")

# Usage
client = LoadBalancedClient([
    "apt_live_key1",
    "apt_live_key2",
    "apt_live_key3"
])

result = client.make_request({
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello"}],
    "api_keys": {"openai": "sk-..."}
})
```

## Request Queueing

Implement a queue to stay within limits:

```python
import queue
import threading
import time

class RateLimitedQueue:
    """Queue requests to respect rate limits."""

    def __init__(self, api_key, requests_per_hour):
        self.api_key = api_key
        self.interval = 3600 / requests_per_hour  # Seconds between requests
        self.queue = queue.Queue()
        self.running = False

    def start(self):
        """Start processing queue."""
        self.running = True
        threading.Thread(target=self._process_queue, daemon=True).start()

    def stop(self):
        """Stop processing queue."""
        self.running = False

    def enqueue(self, payload, callback=None):
        """Add request to queue."""
        self.queue.put((payload, callback))

    def _process_queue(self):
        """Process queued requests with rate limiting."""
        while self.running:
            try:
                payload, callback = self.queue.get(timeout=1)

                # Make request
                response = requests.post(
                    "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
                    headers={"Authorization": f"Bearer {self.api_key}"},
                    json=payload
                )

                result = response.json()

                if callback:
                    callback(result)

                # Wait to respect rate limit
                time.sleep(self.interval)

            except queue.Empty:
                continue
            except Exception as e:
                print(f"Error processing request: {e}")

# Usage
rate_limiter = RateLimitedQueue("apt_live_xyz789", requests_per_hour=1000)
rate_limiter.start()

# Enqueue requests
for i in range(100):
    rate_limiter.enqueue(
        {
            "model": "gpt-4",
            "messages": [{"role": "user", "content": f"Request {i}"}],
            "api_keys": {"openai": "sk-..."}
        },
        callback=lambda r: print(f"Completed: {r['id']}")
    )

# Requests will be processed at maximum safe rate
```

## Monitoring Rate Limits

### Real-Time Dashboard

```python
import requests
import time

def monitor_rate_limits(api_key, interval=60):
    """Monitor rate limit usage in real-time."""
    while True:
        response = requests.get(
            "https://api-aptly.nsquaredlabs.com/v1/me",
            headers={"Authorization": f"Bearer {api_key}"}
        )
        usage = response.json()['usage']

        limit = usage['rate_limit_per_hour']
        current = usage['requests_this_hour']
        remaining = limit - current
        percent = (current / limit) * 100

        # Clear screen and print status
        print("\033[2J\033[H")  # Clear screen
        print("=" * 60)
        print("RATE LIMIT MONITOR")
        print("=" * 60)
        print(f"\nLimit: {limit} requests/hour")
        print(f"Used: {current} ({percent:.1f}%)")
        print(f"Remaining: {remaining}")

        # Visual bar
        bar_length = 50
        filled = int(bar_length * percent / 100)
        bar = "█" * filled + "░" * (bar_length - filled)
        print(f"\n[{bar}] {percent:.1f}%")

        # Status
        if percent >= 90:
            print("\n🚨 CRITICAL: Approaching limit!")
        elif percent >= 75:
            print("\n⚠ WARNING: High usage")
        else:
            print("\n✓ OK: Normal usage")

        time.sleep(interval)

# Usage
monitor_rate_limits("apt_live_xyz789", interval=30)
```

## Best Practices

### 1. Implement Retry Logic

Always handle 429 responses with appropriate retry logic.

```python
# Good: Retry with backoff
if response.status_code == 429:
    wait_until_reset()
    retry()

# Bad: Fail immediately
if response.status_code == 429:
    raise Exception("Rate limited!")
```

### 2. Monitor Usage

Track your usage patterns to predict and prevent rate limits.

### 3. Use Multiple Keys

For high-volume applications, distribute load across multiple API keys.

### 4. Queue Requests

Implement a queue to smooth out traffic spikes.

### 5. Cache Responses

Cache LLM responses when appropriate to reduce API calls.

```python
import hashlib
import json

cache = {}

def cached_request(api_key, payload):
    """Cache LLM responses to reduce API calls."""
    # Create cache key from payload
    cache_key = hashlib.md5(
        json.dumps(payload, sort_keys=True).encode()
    ).hexdigest()

    # Check cache
    if cache_key in cache:
        print("Cache hit!")
        return cache[cache_key]

    # Make request
    response = requests.post(
        "https://api-aptly.nsquaredlabs.com/v1/chat/completions",
        headers={"Authorization": f"Bearer {api_key}"},
        json=payload
    )
    result = response.json()

    # Cache result
    cache[cache_key] = result
    return result
```

## Troubleshooting

### Unexpected Rate Limits

If you're hitting rate limits unexpectedly:

1. **Check all applications** - Multiple apps may use the same key
2. **Review audit logs** - See when requests are made
3. **Check for loops** - Ensure no infinite retry loops
4. **Monitor concurrent requests** - Multiple threads/processes

### Rate Limit Not Resetting

Rate limits reset at the top of each hour (UTC):
- 10:00:00 UTC
- 11:00:00 UTC
- 12:00:00 UTC

If limits don't reset, check system time synchronization.
