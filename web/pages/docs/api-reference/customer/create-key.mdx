---
title: "Create API Key"
api: "POST /v1/api-keys"
description: "Generate a new API key for your account"
---

Create an additional API key for your customer account. Useful for separating keys by environment (dev/staging/prod) or team.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your existing Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Request Body

<ParamField body="name" type="string" required>
  Human-readable name for the key

  Examples: "Production Web App", "Staging Environment", "Mobile App", "Team Backend"
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique key identifier
</ResponseField>

<ResponseField name="key" type="string" required>
  **The full API key** (e.g., `apt_live_xyz789...`)

  <Warning>
    This is the **only time** the full key is returned. Store it securely immediately. It cannot be retrieved again.
  </Warning>
</ResponseField>

<ResponseField name="key_prefix" type="string" required>
  First 12 characters of the key (for identification)
</ResponseField>

<ResponseField name="name" type="string" required>
  The name provided for this key
</ResponseField>

<ResponseField name="rate_limit_per_hour" type="integer" required>
  Requests per hour (inherited from your current API key)
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of key creation
</ResponseField>

## Use Cases

### Create Environment-Specific Keys

Generate separate keys for different environments:

```python
import requests
import os
import json

def create_environment_keys(current_api_key):
    """Create API keys for dev, staging, and prod."""
    environments = ["Development", "Staging", "Production"]
    keys = {}

    for env in environments:
        response = requests.post(
            "https://api-aptly.nsquaredlabs.com/v1/api-keys",
            headers={
                "Authorization": f"Bearer {current_api_key}",
                "Content-Type": "application/json"
            },
            json={"name": f"{env} Environment"}
        )

        if response.status_code == 201:
            result = response.json()
            keys[env.lower()] = result['key']
            print(f"✓ Created {env} key: {result['key_prefix']}")
        else:
            print(f"✗ Failed to create {env} key")

    # Save to .env file or secrets manager
    with open('.env.aptly_keys', 'w') as f:
        for env, key in keys.items():
            f.write(f"APTLY_{env.upper()}_KEY={key}\n")

    print(f"\n✓ Keys saved to .env.aptly_keys")
    print("⚠ Add this file to .gitignore!")

    return keys

# Usage
keys = create_environment_keys("apt_live_xyz789")
```

### Rotate Compromised Key

Replace a compromised key:

```python
def rotate_key(current_api_key, old_key_name, new_key_name):
    """
    Create a new key to replace a compromised one.

    Steps:
    1. Create new key
    2. Update application with new key
    3. Revoke old key
    """
    # Step 1: Create new key
    response = requests.post(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys",
        headers={"Authorization": f"Bearer {current_api_key}"},
        json={"name": new_key_name}
    )

    if response.status_code != 201:
        print("✗ Failed to create new key")
        return None

    new_key = response.json()

    print("✓ New key created!")
    print(f"  New Key: {new_key['key']}")
    print(f"  Key ID: {new_key['id']}")
    print()
    print("⚠ NEXT STEPS:")
    print("  1. Update your application with the new key")
    print("  2. Verify the new key works")
    print("  3. Revoke the old key: DELETE /v1/api-keys/{old_key_id}")

    return new_key

# Usage
new_key = rotate_key(
    "apt_live_xyz789",
    "Compromised Key",
    "Replacement Key - 2026-01-31"
)
```

### Create Team-Specific Keys

Generate keys for different teams:

```python
def create_team_keys(current_api_key, teams):
    """
    Create API keys for multiple teams.

    Args:
        current_api_key: Your current API key
        teams: List of team names
    """
    created_keys = []

    for team in teams:
        response = requests.post(
            "https://api-aptly.nsquaredlabs.com/v1/api-keys",
            headers={"Authorization": f"Bearer {current_api_key}"},
            json={"name": f"{team} Team"}
        )

        if response.status_code == 201:
            result = response.json()
            created_keys.append(result)
            print(f"✓ {team} Team: {result['key_prefix']}")
        else:
            print(f"✗ Failed to create key for {team}")

    # Export to JSON for team distribution
    with open('team_keys.json', 'w') as f:
        json.dump(created_keys, f, indent=2)

    print(f"\n✓ Created {len(created_keys)} keys")
    print("Keys saved to team_keys.json - distribute securely!")

    return created_keys

# Usage
teams = ["Backend", "Frontend", "Mobile", "Data Science"]
keys = create_team_keys("apt_live_xyz789", teams)
```

### Automated Key Creation with Storage

Create a key and automatically store it:

```python
import os
from pathlib import Path

def create_and_store_key(current_api_key, key_name, env_file=".env"):
    """Create a key and append to .env file."""
    # Create the key
    response = requests.post(
        "https://api-aptly.nsquaredlabs.com/v1/api-keys",
        headers={"Authorization": f"Bearer {current_api_key}"},
        json={"name": key_name}
    )

    if response.status_code != 201:
        print(f"✗ Error: {response.json()}")
        return None

    result = response.json()
    new_key = result['key']

    # Append to .env file
    env_var_name = f"APTLY_{key_name.upper().replace(' ', '_')}_KEY"

    with open(env_file, 'a') as f:
        f.write(f"\n{env_var_name}={new_key}\n")

    print(f"✓ Key created and saved!")
    print(f"  Environment Variable: {env_var_name}")
    print(f"  File: {env_file}")
    print(f"  Prefix: {result['key_prefix']}")
    print(f"\n⚠ Make sure {env_file} is in .gitignore!")

    return result

# Usage
create_and_store_key("apt_live_xyz789", "Production", ".env.production")
```

## Rate Limit Inheritance

<Note>
  New keys **inherit the rate limit** from the API key used to create them. You cannot customize the rate limit as a customer - contact your admin if you need different limits.
</Note>

For example:
- If you create a key using an API key with 1000 req/hour limit
- The new key will also have 1000 req/hour limit

## Security Best Practices

### Immediate Storage

<Warning>
  The full API key is only returned **once** during creation. You must:

  1. Capture it from the response immediately
  2. Store it in a secure location (secrets manager, password manager, .env file)
  3. Never commit it to version control
  4. Never share it via insecure channels (email, Slack, SMS)
</Warning>

### Recommended Storage Methods

**Good:**
- ✅ Environment variables (`.env` files in `.gitignore`)
- ✅ Secrets managers (AWS Secrets Manager, HashiCorp Vault)
- ✅ Password managers (1Password, LastPass)
- ✅ CI/CD secrets (GitHub Secrets, GitLab CI Variables)

**Bad:**
- ❌ Hardcoded in source code
- ❌ Committed to Git
- ❌ Sent via email or chat
- ❌ Stored in plain text files in repos

### Naming Conventions

Use descriptive names that help identify the key's purpose:

**Good:**
- "Production Web App - AWS EC2"
- "Staging API - Vercel"
- "Development - Local Machine"
- "Mobile App - iOS"

**Bad:**
- "Key 1"
- "New Key"
- "Test"

## When to Create New Keys

Create new keys when you need to:
- **Separate environments** - Different keys for dev/staging/prod
- **Isolate teams** - Each team has their own key
- **Rotate compromised keys** - Replace leaked or exposed keys
- **Separate applications** - Different apps use different keys
- **Track usage** - Monitor which service is using which key
