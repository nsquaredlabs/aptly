---
title: "PII Statistics"
api: "GET /v1/analytics/pii"
description: "Get PII detection rates, entity type counts, and daily trends"
---

Monitor how often PII is detected in both user inputs and LLM responses. Useful for compliance audits and tuning your [redaction policies](/guides/pii-redaction).

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your Aptly API key: `Bearer apt_live_xxx` or `Bearer apt_test_xxx`
</ParamField>

## Query Parameters

<ParamField query="start_date" type="string">
  Start of the analysis period.

  Formats accepted:
  - ISO 8601: `2026-01-01T00:00:00Z`
  - Date only: `2026-01-01`

  Default: 30 days ago
</ParamField>

<ParamField query="end_date" type="string">
  End of the analysis period.

  Formats accepted:
  - ISO 8601: `2026-01-31T23:59:59Z`
  - Date only: `2026-01-31`

  Default: Current time
</ParamField>

## Response

<ResponseField name="summary" type="object" required>
  High-level PII detection totals for the period

  <Expandable title="summary">
    <ResponseField name="requests_with_input_pii" type="integer">
      Requests where PII was detected in the user's input
    </ResponseField>

    <ResponseField name="requests_with_response_pii" type="integer">
      Requests where PII was detected in the LLM's response
    </ResponseField>

    <ResponseField name="total_requests" type="integer">
      Total requests in the period
    </ResponseField>

    <ResponseField name="input_pii_rate" type="number">
      Percentage of requests that contained input PII
    </ResponseField>

    <ResponseField name="response_pii_rate" type="number">
      Percentage of requests where the LLM response contained PII
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="entity_types" type="array" required>
  Breakdown by PII entity type, sorted by total count descending

  <Expandable title="entity type entry">
    <ResponseField name="type" type="string">
      Entity type (e.g., `PERSON`, `EMAIL_ADDRESS`, `PHONE_NUMBER`, `US_SSN`, `CREDIT_CARD`)
    </ResponseField>

    <ResponseField name="input_count" type="integer">
      Times this entity was found in request inputs
    </ResponseField>

    <ResponseField name="response_count" type="integer">
      Times this entity was found in LLM responses
    </ResponseField>

    <ResponseField name="total_count" type="integer">
      Combined input + response count
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="time_series" type="array" required>
  Daily PII detection trend

  <Expandable title="time series entry">
    <ResponseField name="date" type="string">
      Date (`YYYY-MM-DD`)
    </ResponseField>

    <ResponseField name="requests_with_pii" type="integer">
      Requests on this day with PII in either input or response
    </ResponseField>

    <ResponseField name="total_requests" type="integer">
      Total requests on this day
    </ResponseField>

    <ResponseField name="pii_rate" type="number">
      Percentage of requests with any PII on this day
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="period_start" type="string" required>
  Effective start date of the period (`YYYY-MM-DD`)
</ResponseField>

<ResponseField name="period_end" type="string" required>
  Effective end date of the period (`YYYY-MM-DD`)
</ResponseField>

## Example Requests

<CodeGroup>

```bash cURL
curl "https://api-aptly.nsquaredlabs.com/v1/analytics/pii?start_date=2026-01-01&end_date=2026-01-31" \
  -H "Authorization: Bearer apt_live_xyz789"
```

```python Python
import requests

response = requests.get(
    "https://api-aptly.nsquaredlabs.com/v1/analytics/pii",
    headers={"Authorization": "Bearer apt_live_xyz789"},
    params={"start_date": "2026-01-01", "end_date": "2026-01-31"}
)

data = response.json()
print(f"Input PII rate:    {data['summary']['input_pii_rate']}%")
print(f"Response PII rate: {data['summary']['response_pii_rate']}%\n")

for entity in data["entity_types"]:
    print(f"  {entity['type']}: {entity['total_count']} detections "
          f"(input {entity['input_count']}, response {entity['response_count']})")
```

```javascript JavaScript
const response = await fetch(
  'https://api-aptly.nsquaredlabs.com/v1/analytics/pii?' +
  new URLSearchParams({ start_date: '2026-01-01', end_date: '2026-01-31' }),
  { headers: { 'Authorization': 'Bearer apt_live_xyz789' } }
);

const data = await response.json();
console.log(`Input PII rate: ${data.summary.input_pii_rate}%`);
data.entity_types.forEach(e => {
  console.log(`  ${e.type}: ${e.total_count} (in ${e.input_count}, resp ${e.response_count})`);
});
```

</CodeGroup>

## Notes

- A single request can count toward both `requests_with_input_pii` and `requests_with_response_pii` if PII is found on both sides.
- `entity_types` counts individual detections, not unique requests. One message containing two names adds 2 to `PERSON` `input_count`.
- The `time_series` `pii_rate` treats a request as having PII if it was detected in *either* input or response.
